# 用户操作手册

> 面向运营/教研人员的系统使用指南。本系统帮助你将 1 对 1 课程辅导的录音文字转化为社交媒体内容、体系化课程与可复用 Skill 库。

---

## 界面概览

打开浏览器访问 `http://localhost:3000`，进入系统管理后台：

- **左侧边栏**：导航菜单（录音稿 / 社媒内容 / Skill 库 / 课程体系）
- **顶部区域**：面包屑路径 + 当前页面标题
- **主内容区**：数据表格、卡片列表或操作表单

> 移动端（<768px）时，侧边栏自动收起，通过左上角汉堡菜单展开。

---

## 第一章：上传录音稿

### 1.1 准备文件

系统接受以下格式的录音文字稿：
- **TXT 文件**（`.txt`）
- **Markdown 文件**（`.md`、`.markdown`）

**文件要求**：
- 编码必须为 **UTF-8**
- 大小不超过 **20MB**
- 内容不能为空

> 建议在上传前检查转写文本是否包含大量时间戳标记（如 `[00:12:34]`），系统会自动清洗这些标记，无需手动删除。

### 1.2 上传步骤

1. 点击左侧边栏 **「上传录音稿」**，进入上传页面。
2. 在页面中央的拖拽区域，将文件拖入，或点击选择文件。
3. 填写 **标题**（如：「2026-05-11 小明同学第3期辅导」）。
4. 点击 **「开始上传」** 按钮。

「此处为上传页面截图：拖拽区域、标题输入框、上传按钮」

5. 等待上传进度条完成。上传成功后，系统会自动跳转至该录音稿的详情页。

### 1.3 查看录音稿列表

1. 点击左侧边栏 **「录音稿管理」**。
2. 列表以表格形式展示所有已上传的录音稿，包含：标题、文件名、字数、状态、上传时间。
3. 使用顶部的搜索框可按标题或内容关键词筛选。
4. 点击状态标签（待处理 / 处理中 / 已完成 / 失败）可快速过滤。

「此处为录音稿列表页面截图：表格、搜索框、状态筛选按钮」

### 1.4 录音稿详情页

点击任意录音稿标题进入详情页：
- **原始文本**：可折叠/展开查看完整录音文字
- **内容片段**：系统预处理后的高价值洞察（按主题分类）
- **操作按钮**：可触发三个 AI 工作流（社媒生成 / Skill 提取 / 课程生成）

「此处为录音稿详情页面截图：文本折叠区域、内容片段卡片、工作流触发按钮」

## 第二章：生成社交媒体内容

### 2.1 触发社媒生成

1. 进入目标录音稿的 **详情页**。
2. 点击 **「生成社媒内容」** 按钮。
3. 系统状态变为 **处理中**，此过程通常需要 10–60 秒（取决于录音长度与 LLM 响应速度）。
4. 处理完成后，录音稿状态变为 **已完成**。

> 同一篇录音稿在处理中时，无法再次触发任何工作流，请等待完成后再操作。

### 2.2 查看与编辑社媒草稿

1. 点击左侧边栏 **「社媒内容」**。
2. 页面以卡片形式展示所有生成的社交媒体草稿，每张卡片包含：
   - 标题（含 emoji 风格）
   - 正文预览
   - 标签
   - 配图建议
   - 生成来源（关联的录音稿标题）

「此处为社媒内容页面截图：卡片列表、编辑按钮、复制按钮」

3. 点击卡片上的 **「编辑」** 按钮，可在弹窗中修改：
   - 标题
   - 正文
   - 标签（以逗号分隔）
   - 发布状态（草稿 / 已发布）
4. 点击 **「复制 Markdown」** 一键复制完整内容，直接粘贴至小红书或其他平台编辑器。

### 2.3 搜索与筛选

- 使用顶部搜索框按标题或正文关键词查找草稿。
- 按状态筛选查看「已发布」或「草稿」内容。

## 第三章：查看 Skill 库

### 3.1 Skill 库页面

1. 点击左侧边栏 **「Skill 库」**。
2. 表格展示所有已提取的 Skill 条目，每行包含：
   - **场景**：该 Skill 适用的面试/求职场景
   - **动作**：具体应执行的技巧或方法
   - **预期结果**：执行后应达到的效果
   - **视角**：`面试官` 或 `候选人`
   - **创建时间**

「此处为 Skill 库页面截图：表格、视角筛选、搜索框」

### 3.2 筛选与搜索

- **视角筛选**：点击顶部「面试官」或「候选人」按钮，仅显示对应视角的 Skill。
- **全文搜索**：输入关键词（如「自我介绍」「STAR」），系统会跨场景/动作/预期结果进行语义搜索，返回最相关的结果。
- **详情弹窗**：点击任意 Skill 行，可查看完整内容及其来源录音稿片段。

### 3.3 Skill 的去重与合并

系统会自动对相似度 >85% 的 Skill 进行合并：
- 合并时，重复条目的场景/动作/预期结果会被整合到保留条目中
- 被合并的记录会在保留条目的详情中标注来源 ID
- 你无需手动操作，但可在详情弹窗中查看合并历史

## 第四章：生成课程

### 4.1 手动生成课程

1. 点击左侧边栏 **「课程体系」**。
2. 点击页面右上角的 **「生成新课程」** 按钮。
3. 在弹窗中填写 **主题标签**（可选，如「面试」「简历」「薪资谈判」），用于过滤关联的内容片段。
4. 点击 **确认**，系统开始生成课程。

「此处为课程体系页面截图：课程卡片列表、生成新课程弹窗」

### 4.2 自动触发机制

当系统中所有录音稿的累计 processed 字数超过 **50,000 字** 时，课程生成工作流支持自动触发。手动生成不受此限制。

### 4.3 查看课程详情

1. 在课程列表中点击任意课程卡片。
2. 弹窗展示课程结构：
   - 课程标题与简介
   - 课时列表（多级模块 + 课时树形展开）
   - 每节课的：标题、大纲、详细讲义内容

「此处为课程详情弹窗截图：树形课时结构、讲义内容展开」

> 课程生成依赖历史录音稿中 `score >= 3` 的内容片段。如果生成结果不理想，建议上传更多高质量录音稿后重新生成。

## 常见问题 FAQ

### Q1：上传文件时提示「文件编码错误」怎么办？

确保文件保存为 **UTF-8** 编码。常见解决方法：
- Windows 记事本：另存为 → 编码选择「UTF-8」
- VS Code：点击右下角编码标识 → 选择「Save with Encoding」→ UTF-8
- 命令行：`iconv -f GBK -t UTF-8 input.txt > output.txt`

### Q2：上传后录音稿状态一直是「处理中」？

正常情况下，只有触发 AI Pipeline（社媒/Skill/课程）后状态才会变为 `processing`。如果上传后长期显示 `processing`：
1. 刷新页面查看最新状态
2. 检查后端服务是否正常运行：`curl http://localhost:8000/health`
3. 查看 Docker 日志：`docker-compose logs -f backend`

### Q3：生成的小红书内容不满意，如何重新生成？

当前版本暂不支持对同一篇录音稿直接「重新生成」。建议：
1. 在 **社媒内容** 页面编辑现有草稿，手动修改标题和正文
2. 或上传同一期辅导的补充/修正版录音稿，再次触发社媒生成

### Q4：课程生成失败，提示「没有足够的高质量内容片段」？

原因：系统中可用于聚类生成课程的内容片段数量不足或质量分过低。
解决：
1. 多上传几期录音稿并触发 **社媒生成** 或 **Skill 提取**（这两个工作流会产生 `ContentFragment`）
2. 确保录音稿内容丰富，避免过短的转写文本

### Q5：Skill 库中出现了重复条目？

系统会在 Skill 提取时自动合并相似度 >85% 的条目。如果你仍看到相似内容：
1. 确认它们是否来自不同视角（面试官 vs 候选人）——不同视角的 Skill 不会被合并
2. 确认它们的场景/动作是否确有差异——语义不同但字面相似的 Skill 可能未被识别为重复
3. 如需手动清理，可联系开发者通过数据库操作删除

### Q6：前端页面打不开或显示空白？

1. 确认前端容器已启动：`docker-compose ps`
2. 检查浏览器控制台是否有网络错误——确认 `VITE_API_BASE_URL` 指向正确的后端地址
3. 尝试强制刷新：`Ctrl + F5`（Windows）或 `Cmd + Shift + R`（Mac）

### Q7：如何备份系统中的数据？

数据库与向量库数据均通过 Docker Volume 持久化：
```bash
# 备份 PostgreSQL
docker-compose exec postgres pg_dump -U appuser audioprofit > backup_$(date +%Y%m%d_%H%M%S).sql

# 备份 Qdrant（直接复制 volume 数据）
docker run --rm -v audio-profit_qdrant_data:/data -v $(pwd):/backup alpine tar czf /backup/qdrant_backup.tar.gz -C /data .
```
